## 1. 크롤러 0건 수집 시 처리

| 항목 | 내용 |
|------|------|
| **상태** | ✅ 완료 |
| **결정** | C — 연속 5회 0건이면 알림 발송 |
| **구현** | `CrawlerHealthService` (`services/crawler_health_service.py`), `EmailService.send_alert()` |

**배경:** 사이트 구조가 바뀌면 크롤러가 조용히 0건을 반환한다. 현재는 0건이어도 에러로 처리되지 않고 "발송할 새 이벤트 없음"으로 끝난다.

**선택지:**
- A. 0건이면 에러 로그만 남기고 그냥 종료 (현재 동작)
- B. 0건이면 관리자에게 별도 알림 발송
- C. 연속 N회 0건이면 알림 발송 ← **채택**

## 3. 스케줄러: cron vs launchd vs pmset

| 항목 | 내용 |
|------|------|
| **상태** | 🔲 미결 |
| **결정** | — |

**배경:** macOS cron은 스케줄 시각에 Mac이 절전 상태이면 해당 job을 **스킵**한다. 실제로 9시 job이 실행되지 않는 현상이 발생했음 (시스템 로그에서 확인).

**선택지:**

- A. **launchd 전환** — `StartCalendarInterval` 사용 시 절전 중 missed job을 깨어난 후 실행. 단, 정확한 시간(9시, 15시) 보장 안 됨. 10시에 깨어나면 10시에 이메일 발송.
- B. **pmset + cron** — `pmset repeat`으로 Mac을 지정 시각에 자동으로 깨운 뒤 cron 실행. 정확한 시간 보장되나 `pmset repeat`은 시간 하나만 등록 가능 → 9시, 15시 두 번 모두 적용 불가.
- C. **현행 유지 (cron)** — Mac을 절전 없이 항상 켜두는 경우에만 안정적.

**트레이드오프:**

| | 정확한 시간 | 하루 2회 보장 | 설정 난이도 |
|---|---|---|---|
| A. launchd | 절전 시 지연 | O | 보통 |
| B. pmset + cron | O (1회만) | X | 보통 |
| C. cron 유지 | O (Mac 켜 있을 때만) | 절전 시 스킵 | 쉬움 |

## 2. 크롤러 픽스처(fixture) 관리 주기

| 항목 | 내용 |
|------|------|
| **상태** | ✅ 완료 |
| **결정** | A — 수동으로 주기적으로 픽스처 갱신 (현재 방식 유지) |
| **이유** | 크롤러가 연속 5회 0건 알림(항목 1)으로 구조 변경을 감지하므로 별도 자동화 불필요 |

**배경:** 테스트 픽스처(HTML/JSON)가 실제 사이트와 달라지면 테스트는 통과하지만 실제 크롤링은 실패한다. Megabox/Lotte Cinema에서 실제로 발생했음.

**선택지:**
- A. 수동으로 주기적으로 픽스처 갱신 (현재 방식) ← **채택**
- B. CI에서 실제 사이트 구조를 검증하는 smoke test 추가
- C. 크롤러 통합 테스트를 별도 스크립트로 분리
