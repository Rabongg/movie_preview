# Movie Preview Alarm v3 - 요구사항 정의서

## 1. 프로젝트 개요

CGV, Megabox, Lotte Cinema에서 영화 무대인사/시사회 정보를 크롤링하여 하루 2회 이메일로 알림을 발송하는 Python 기반 자동화 서비스.

v3의 핵심 목표: DB 없이 JSON 파일만으로 중복 발송을 방지하고, Python 단독으로 동작하는 경량 구조.

---

## 2. 기능 요구사항

### 2-1. 크롤링

| 항목 | 내용 |
|------|------|
| 대상 사이트 | CGV, Megabox, Lotte Cinema |
| 수집 대상 | 무대인사, 시사회(GV 포함) |
| 수집 항목 | 영화 제목, 행사 날짜/시간, 극장명, 참석 배우/감독, 예매 링크 |
| 크롤링 방식 | Selenium (JavaScript 렌더링 필요한 경우) 또는 requests + BeautifulSoup |

### 2-2. 이메일 발송

| 항목 | 내용 |
|------|------|
| 발송 시간 | 오전 9시, 오후 3시 (환경 변수로 변경 가능) |
| 발송 방식 | Gmail SMTP (SSL) |
| 수신자 | 복수 이메일 지원 (`,` 구분으로 환경 변수에 설정) |
| 이메일 형식 | HTML 포맷 |
| 포함 정보 | 영화 제목, 행사 유형(무대인사/시사회), 날짜/시간, 극장, 배우/감독, 예매 링크 |
| 발송 조건 | 이전에 발송한 적 없는 이벤트만 발송. 새 이벤트가 없으면 발송 안 함 |

### 2-3. 중복 방지

| 항목 | 내용 |
|------|------|
| 저장 방식 | JSON 파일 (`data/sent_events.json`) |
| 중복 판단 기준 | `극장 + 영화 제목 + 행사 날짜 + 행사 유형` 조합의 해시값 |
| 저장 내용 | 이벤트 ID, 제목, 극장, 날짜, 발송 시각 |

### 2-4. JSON 파일 아카이브 관리

| 항목 | 내용 |
|------|------|
| 아카이브 단위 | 월별로 `data/archive/YYYY-MM.zip`으로 압축 |
| 아카이브 조건 | 전달(前月)이 끝나면 해당 월 데이터를 zip으로 묶어 보관 |
| 삭제 조건 | 생성된 지 2개월이 지난 zip 파일 자동 삭제 |
| 실행 시점 | `main.py` 실행 시마다 아카이브 정리 로직 함께 실행 |

### 2-5. 스케줄 실행

| 항목 | 내용 |
|------|------|
| 실행 환경 | 로컬 PC |
| 스케줄 방식 | macOS: crontab / Windows: Task Scheduler |
| 실행 주기 | 하루 2회 (시간은 환경 변수로 설정) |

### 2-6. 로깅

| 항목 | 내용 |
|------|------|
| 출력 대상 | 콘솔 + 파일 동시 출력 |
| 로그 파일 위치 | `logs/app.log` |
| 로테이션 | 파일 크기 기준 RotatingFileHandler (최대 5MB, 백업 3개) |
| 로그 레벨 | INFO (기본), ERROR (크롤링/발송 실패 시) |
| 포함 정보 | 실행 시각, 크롤링 결과 수, 새 이벤트 수, 발송 성공/실패 여부 |

## 3. 비기능 요구사항

- **단순성**: 서버, DB, 별도 인프라 없이 동작
- **이식성**: `.env` 파일 하나로 모든 설정 관리
- **확장성**: 새로운 극장 크롤러를 쉽게 추가할 수 있는 구조
- **안정성**: 특정 극장 크롤링 실패 시 다른 극장 발송은 정상 진행
- **가독성**: 이메일은 HTML 형식으로 보기 좋게 포맷
- **테스트 가능성**: 외부 의존성(크롤링, 이메일)을 mock으로 격리하여 단위 테스트 가능한 구조

## 4. 테스트 요구사항

| 항목 | 내용 |
|------|------|
| 테스트 프레임워크 | pytest |
| 개발 방식 | TDD (테스트 먼저 작성 후 구현) |
| 크롤러 테스트 방식 | Mock HTML fixture 파일 사용 (실제 네트워크 요청 없음) |
| 이메일 테스트 방식 | `smtplib.SMTP_SSL` mock 처리 |
| 커버리지 측정 | pytest-cov 사용 |

### 테스트 대상별 전략

| 모듈 | 테스트 방식 | 비고 |
|------|------------|------|
| `models/event.py` | 순수 단위 테스트 | event_id 해시 일관성 검증 |
| `services/storage_service.py` | 임시 디렉토리 사용 | 실제 파일 I/O, 아카이브 로직 포함 |
| `services/email_service.py` | smtplib mock | 실제 메일 발송 없음 |
| `crawlers/*.py` | HTML fixture mock | 각 극장별 HTML 파일 저장 후 파싱 테스트 |
| `main.py` | 통합 테스트 | 크롤러 + 스토리지 + 이메일 흐름 검증 |

## 5. 환경 변수 명세 (`.env`)

```env
# 발신자
SENDER_EMAIL=your_email@gmail.com
SENDER_KEY=your_gmail_app_password

# 수신자 (복수 시 쉼표로 구분)
RECEIVER_EMAILS=recipient1@example.com,recipient2@example.com

# 발송 시간 (24시간 형식)
SEND_HOUR_MORNING=9
SEND_HOUR_AFTERNOON=15
```

## 6. 제외 범위

- 사용자 로그인/회원 관리 기능 없음
- 웹 서버/API 서버 없음
- DB (MySQL, Redis 등) 사용 안 함
- 푸시 알림, SMS 등 이메일 외 채널 없음
